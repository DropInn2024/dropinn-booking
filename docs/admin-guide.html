01
系統架構概覽
在開始建置之前，了解整個系統的組成

雫旅訂房系統由三個主要技術層構成，全部部署在 Cloudflare 平台上，不需要自架伺服器。

技術架構
層技術說明
前端Cloudflare PagesHTML/CSS/JS 靜態頁面，從 GitHub 自動部署
後端 APICloudflare WorkersEdge Runtime，處理所有 API 請求
資料庫Cloudflare D1SQLite 資料庫，儲存訂單、帳號、房務資料
備份Cloudflare R2每週自動備份資料庫

每位客戶的部署方式

目前系統為單租戶架構（Single-tenant）：每位購買者擁有獨立的 Cloudflare 帳號與資料庫，資料完全隔離。你（賣家）負責幫他們建置並交機。

架構說明
客戶 A 的資料庫、客戶 B 的資料庫彼此獨立，互不可見。你只是幫他們設定好，設定完成後的資料都只屬於他們自己。

一次性平台設定
你自己的開發環境，只需要做一次
注意
這一章是給你（賣家）準備自己電腦環境的步驟。如果你已經有可以正常 wrangler deploy 的環境，可以直接跳到第三章。

需要的帳號與工具

項目說明
Cloudflare 帳號你自己的 Cloudflare 帳號，用來幫客戶建置
GitHub 帳號存放程式碼，Pages 從這裡自動部署
Node.js版本 18 以上，用來執行 Wrangler
Wrangler CLICloudflare 官方部署工具
Resend 帳號寄送訂單確認信、提醒信

安裝 Wrangler

# 安裝 Wrangler
npm install -g wrangler

# 登入 Cloudflare（會開瀏覽器）
wrangler login

git clone git@github.com:你的帳號/dropinn-客戶名稱.git
3
進入 worker 目錄，安裝依賴
cd worker && npm install
03
新客戶開通流程
每接一位新客戶，依序執行以下步驟

Step 1｜向客戶收集資訊

開始建置前，先向客戶確認以下資訊：

客戶資料表
欄位說明範例
民宿名稱顯示在後台頂部海風小屋
後台帳號登入 notforyou 後台用haifeng2026
後台密碼建議 8 碼以上英數混合（客戶自訂）
房務密碼登入 restoretheblank 房務介面（客戶自訂）
網域客戶的自訂網域（選填）haifeng.tw
聯絡信箱訂單通知、提醒信收件匣owner@haifeng.tw
Resend API Key寄信服務，客戶自行申請re_xxxx

Step 2｜建立 D1 資料庫

# 建立新資料庫（客戶名稱作為識別）
wrangler d1 create dropinn-客戶名稱

# 記下輸出的 database_id，稍後填入 wrangler.toml
✅  Created database 'dropinn-haifeng'
{
  "database_id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
}

Step 3｜更新 wrangler.toml

修改 worker/wrangler.toml，填入剛才取得的 database_id：

name = "dropinn-haifeng-worker"

[[d1_databases]]
binding = "DB"
database_name = "dropinn-haifeng"
database_id = "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"

routes = [
  { pattern = "haifeng.tw/api/*", zone_name = "haifeng.tw" }
]

Step 4｜執行資料庫 Migration

# 初始化資料表結構
cd worker
wrangler d1 migrations apply dropinn-客戶名稱 --remote

Step 5｜設定 Wrangler Secrets

將客戶的帳號密碼與 API 金鑰設為加密的 secrets（不會存在程式碼裡）：

# 後台登入帳號
wrangler secret put ADMIN_LOGIN_ID
› 輸入：haifeng2026

# 後台密碼（先用工具轉成 hash，見下方說明）
wrangler secret put ADMIN_PASSWORD_HASH
› 輸入：（密碼的 SHA-256 hash）

# 房務密碼
wrangler secret put HOUSEKEEPING_PASSWORD
› 輸入：（房務人員的密碼）

# JWT 簽名密鑰（隨機產生一串長字串）
wrangler secret put TOKEN_SECRET
› 輸入：（例如 openssl rand -hex 32 產生的字串）

# 寄信 API Key
wrangler secret put RESEND_API_KEY
› 輸入：re_xxxxxxxx

# 訂單通知信箱
wrangler secret put ADMIN_NOTIFY_EMAIL
› 輸入：owner@haifeng.tw

# 雜湊用的 salt（隨機字串）
wrangler secret put SALT
› 輸入：（openssl rand -hex 16）
密碼雜湊說明
系統儲存的是密碼的 hash，不是明文。設定 ADMIN_PASSWORD_HASH 前，需先用系統提供的小工具產生 hash。在終端機執行：

node -e "require('crypto').createHash('sha256').update('SALT' + 'PASSWORD').digest('hex') |> console.log"

或使用 worker 目錄裡的 scripts/hash-password.js 輔助工具。

Step 6｜部署 Worker

# 部署後端 API
cd worker
wrangler deploy

Step 7｜設定 Cloudflare Pages

1
Cloudflare Dashboard → Pages → Create a project
選擇「Connect to Git」，連結客戶的 GitHub repo。
2
Build 設定
Build command 留空（純靜態），Output directory 填 /（根目錄）。
3
綁定自訂網域
Pages → Custom domains → 填入客戶的網域（如 haifeng.tw）。Cloudflare 會自動設定 DNS 與 SSL。
截圖位置
Cloudflare Pages 建立專案的設定畫面

Step 8｜更新 config.public.js

在前端根目錄編輯 config.public.js，填入客戶的網域：

window.DROPINN_CONFIG = {
  API_BASE: 'https://haifeng.tw',
  RECAPTCHA_SITE_KEY: '客戶的 reCAPTCHA Site Key'
};

Step 9｜驗收測試

1
開啟訂房首頁，選一組日期並送出測試訂單
確認可以看到確認畫面，客戶信箱收到通知信。
2
登入 notforyou 後台
前往 haifeng.tw/notforyou/，輸入帳號密碼，確認登入成功並能看到測試訂單。
3
登入房務介面
前往 haifeng.tw/restoretheblank/，確認房務日曆正常顯示。

05
日常維護與更新
系統上線後的維護流程

前端更新（推 GitHub 即自動生效）

# 在客戶的 repo 目錄下
git add .
git commit -m "說明改了什麼"
git push

Cloudflare Pages 監聽 GitHub，push 後約 1 分鐘自動部署完成。

後端更新（Worker）

# 需要手動重新部署 Worker
cd worker
wrangler deploy
什麼時候需要重新 deploy Worker？
當你修改了 worker/src/ 底下的任何 .js 檔案時，都需要重跑 wrangler deploy。純前端的 HTML/CSS/JS 改動只需要 push 到 GitHub 即可。

資料庫備份

系統已設定每週日台灣時間上午 10:00 自動將資料庫備份到 Cloudflare R2。如需手動備份：

wrangler d1 export dropinn-客戶名稱 --remote --output backup-YYYY-MM-DD.sql
06
常見問題排查
遇到問題時的排查方向

後台登入失敗

症狀可能原因處理方式
帳號密碼錯誤ADMIN_PASSWORD_HASH 設定有誤重新計算 hash，用 wrangler secret put 更新
一直跳回登入頁TOKEN_SECRET 未設定確認 wrangler secret list 有 TOKEN_SECRET
API 回傳 500資料庫 migration 未執行重跑 wrangler d1 migrations apply

訂單通知信收不到

1
確認 RESEND_API_KEY 正確
到 Resend dashboard 確認 API key 有效，且網域已驗證。
2
確認 ADMIN_NOTIFY_EMAIL 設定
執行 wrangler secret list 確認 ADMIN_NOTIFY_EMAIL 存在。
3
查看 Worker 日誌
wrangler tail 可即時看到 Worker 的 console.log 輸出，有助於診斷錯誤。

同業日曆讀不到資料

常見原因
業者帳號的 approvalStatus 不是 approved，或 isActive 不是 1。可在後台管理介面核准帳號，或直接用 D1 console 更新。
