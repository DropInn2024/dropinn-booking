# 雫旅：狀態說法整理 ＋ 網頁清單與網域

## 一、狀態是「我們／同業／系統」用的，給客人的要變成說法

### 1. 四種狀態 → 給客人的說法（已套用）

| 系統狀態 | 用途 | 給客人的說法（信件／網頁，已套用） |
|----------|------|------------------------------------|
| **洽談中** | 客人剛下單、尚未付訂 | 信主旨：「預約申請已收到」；內文：「您的預約申請已收到」「請於 48 小時內與我們聯繫確認」 |
| **已付訂** | 已收訂金、訂單成立 | 信主旨：「訂單成立」；內文：「訂單成立」「您的預約資訊」 |
| **取消** | 訂單取消 | 有訂金：主旨「已為您辦理退訂與退款說明」、內文「已為您辦理退訂，訂單 xxx 的退款已辦理」；無訂金：主旨「謝謝您，xxx」、感謝文 |
| **完成** | 入住結束 | **退房感謝信**：已設置於 `website/post-stay-thankyou.html`（標題「島嶼的餘韻」）。若之後要自動寄出，可將該頁 HTML 接到 `EmailTemplates.js` 的退房模板並在狀態改為「完成」時觸發。 |

**小結**：系統只使用「洽談中、已付訂、取消、完成」；給客人的信／網頁一律用上表說法，不出現這四個狀態詞。

## 三、全部用 dropinn.tw 網域、不露出 GitHub 的做法

### 1. 靜態站（訂房首頁、約定、空房、旅遊手冊、退房感謝、同業、後台入口）

- 把 **dropinn.tw** 指到您現在放靜態檔的地方（例如 **GitHub Pages 自訂網域** 或 **Cloudflare Pages**）。
- 這樣所有「網頁」的網址都是 `https://dropinn.tw/...`，客人與同業**不會看到 github.io 或 github.com**。

### 2. 後台／房務「入口」網址

- **做法 A（建議）**：把 `https://dropinn.tw/admin`、`https://dropinn.tw/housekeeping` 設成**轉址**到 GAS 的 `?page=admin`、`?page=housekeeping`。  
  - 例如：`dropinn.tw/admin` → 301 轉到 `https://script.google.com/.../exec?page=admin`  
  - 使用者只會記得 dropinn.tw/admin，實際連到的是 Google，**不會看到 GitHub**。
- **做法 B**：維持從靜態站載入 `admin.html`／`housekeeping.html`（`dropinn.tw/admin.html`），頁面再打 GAS API；一樣不會出現 GitHub。

### 3. 信件與連結裡的網址（已改為 dropinn.tw）

程式內已改為使用 **dropinn.tw**：

| 檔案 | 網址（已套用） |
|------|----------------|
| `EmailTemplates.js` | 雫旅約定連結：`https://dropinn.tw/website/agreement.html` |

寄給客人的信裡的「雫旅約定」連結已為 dropinn.tw，不會出現 github.io。

### 4. GAS 後端

- API 與後台頁面由 **Google Apps Script** 提供，網址本來就是 `script.google.com`，**與 GitHub 無關**，無需改網域。
