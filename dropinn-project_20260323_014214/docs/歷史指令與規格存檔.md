## 一、一頁式官網設計指令（歷史草稿，尚未在本 repo 完整實作）

來源：早期給 Cursor 的網站設計任務  
狀態：**僅供參考（歷史草稿）** — 目前專案主線以 GAS 訂房系統 + 現有 `index.html` 為主，此規格可留作未來重做官網時的設計藍本。

### 1. 設計系統（節錄）

- 極簡主義 + 侘寂美學，留白大、字距大、陰影極少  
- 字體：
  - 英文：`'Cormorant Garamond', serif`
  - 中文：`'Noto Serif TC', serif`（300, 400）
- 色彩計畫：
  - `--cream: #f8f6f0;` 背景
  - `--ink: #332c27;` 主文字
  - `--stone: #b5aba0;` 輔助
  - `--accent: #b8795a;` 點綴
- 設計原則（關鍵）：
  - 區塊間距 ≥ 160px，行距 ≥ 1.8
  - 標題 letter-spacing 0.2–0.3em
  - 動畫 `transition: 0.6s cubic-bezier(0.4, 0, 0.2, 1)`

### 2. 區塊架構（摘要）

- Navbar：右上固定 `[空間] [關於雫旅]`，smooth scroll  
- Hero：滿版 100vh，中央文案「DROP INN, DROP YOUR WORRIES / 雫旅 / 放下行李，也放下節奏…」  
- Gallery：Masonry Grid + Lightbox  
- About：一整段「一滴水／那一滴時光」文案  
- House Rules：入住默契 01–04 條列＋「查看完整訂房約定」按鈕  
- Booking：自製日曆 + 範圍選取 + Modal 表單（包棟規模、加床、聯絡資訊、預估金額）  
- Footer：IG/LINE、地址、版權

### 3. 技術要求（當時規劃）

- 純 HTML + CSS + Vanilla JS（可用 Day.js）  
- 響應式：桌機／平板／手機三斷點  
- 動畫：smooth scroll、Intersection Observer 淡入、Modal 動畫、日曆互動  
- 日曆假資料 + 價格表（3/4/5 間房 + 加床）

> 備註：目前 repo 的 `index.html` 走的是「訂房表單 + 動畫 Hero」路線，**並未完全依照這份一頁式設計稿落實**；未來若要重做官網，可直接以此章節為設計 brief。

## 二、雙 API 部署與上線流程（已部分整合，細節以 DEPLOYMENT.md 為準）

來源：早期《雫旅訂房系統 - 雙 API 部署說明》和《上線部署流程》  
狀態：**現行版本已整合** — 具體步驟與 API 清單已拆入 `DEPLOYMENT.md` 與 `docs/GAS腳本功能與使用.md`，本節保留概念與架構。

### 1. Public / Admin 雙 API 架構

- **Public API（訂房 / 公開）**
  - 給前台 `index.html` 和 `availability.html` 使用
  - 支援：`createBooking`、`getBookedDates`、`checkAvailability` 等
  - 有 reCAPTCHA、基本 rate limit

- **Admin API（後台 / 房務）**
  - 給 `admin.html`、`housekeeping.html` 使用
  - 包含：查詢全部訂單、更新訂單、日曆重建、清理老資料、財務統計等
  - 透過 `ADMIN_API_KEY` + `isAdminAction()` 控制權限

- 部署建議：**同一個 GAS 專案、兩個 Web App 部署**（Public / Admin 各一 URL）
  - Public 給靜態網頁（GitHub Pages / Netlify）
  - Admin 僅你使用，不公開

### 2. 部署與上線流程（概念）

- Script Properties 設定（機密）：
  - `SHEET_ID`、`RECAPTCHA_SECRET_KEY`、Email 寄件者、Calendar ID 等
- 上線檢查清單：
  - 價格計算、入住時間、日曆同步、Email 通知、取消政策文案
- 前端部署：
  - `index.html` / `availability.html` / `agreement.html` 等推到 GitHub Pages
  - `config.public.js` 中設定 `API_URL` / `API_URL_PUBLIC`

> 實際、可執行的步驟與指令已收斂在 `DEPLOYMENT.md`；若兩者不一致，以 `DEPLOYMENT.md` + 現行程式碼為準。
